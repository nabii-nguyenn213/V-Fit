                        CONFIGURATION ARCHITECTURE
================================================================================

NEW CONFIGURATION SYSTEM:

Priority Hierarchy:
  1. Environment Variable (Highest - ENVIRONMENT=production)
  2. Config Class (Dev/Staging/Prod specific)
  3. Default Value (Lowest - if not set elsewhere)

Three Environment Configurations:

A. DevelopmentConfig
   - DEBUG = True
   - LOG_LEVEL = "DEBUG"
   - CORS_ORIGINS = "*" (allow all)
   - Purpose: Local development with debugging

B. StagingConfig
   - DEBUG = False
   - LOG_LEVEL = "INFO"
   - CORS_ORIGINS = restricted (from env var)
   - Purpose: Pre-production testing

C. ProductionConfig
   - DEBUG = False
   - LOG_LEVEL = "WARNING"
   - CORS_ORIGINS = restricted (from env var, required)
   - SECRET_KEY = required (will raise error if not set)
   - GEMINI_API_KEY = required (will raise error if not set)
   - Purpose: Maximum security for production

Key Features:
  - Same codebase works in all environments
  - No secrets in git (env vars only)
  - Easy to scale horizontally
  - Production is strict, development is permissive
